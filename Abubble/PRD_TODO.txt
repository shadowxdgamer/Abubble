1. Product Requirements Document (PRD)
Product Name: ABoard (Overlay)
Type: Android Accessibility Service & System Overlay
Core Value: A "Brain Extension" for your existing keyboard. It reads what you are typing in any app (WhatsApp, Slack, Email), processes it with AI, and pastes the improved version back automatically.

High-Level User Flow (The "Magic")
The Trigger: A small, semi-transparent "Brain" bubble sits on the edge of the screen (like a Messenger Chat Head).

Activation:

Tap: Expands a small "Prompt Box" floating right over your current app.

Context: The app instantly "reads" the text field you were focusing on (e.g., the email draft).

The Interaction:

User Types: "Make this more professional" or "Fix grammar."

AI Processing: Calls OpenRouter (with DeepSeek/Reasoning support if enabled).

The Result: The prompt box vanishes, and the AI's text is automatically pasted into the original text field, replacing or appending the text.

Key Features
BYOK (Bring Your Own Key): Secure storage for OpenRouter API Key.

Floating UI: Built using WindowManager, independent of the app you are currently using.

Context Awareness: Uses AccessibilityNodeInfo to grab text from the focused input field.

DeepSeek Reasoning:

Detects if the selected model supports "reasoning" (via GET /models).

Shows a "Thinking" toggle in the floating prompt box.

Supports multi-turn refinement (e.g., "Actually, make it shorter").

2. Technical Architecture
A. The Service (MindBubbleService)
Type: AccessibilityService.

Capabilities:

retrieveWindowContent: To read screen text.

performGlobalAction: To trigger Paste.

SYSTEM_ALERT_WINDOW: To draw UI over other apps.

B. The Data Layer (OpenRouter)
Endpoint: POST https://openrouter.ai/api/v1/chat/completions

Payload Logic:

System Prompt: "You are a writing assistant. The user is typing in [App Name]. Current text: '[Context]'. Task: [User Prompt]."

Reasoning Toggle: If enabled, inject {"reasoning": {"enabled": true}}.

C. The UI Layer (WindowManager)
View 1 (The Bubble): A small ImageView with a touch listener for dragging and tapping.

View 2 (The Prompt Dialog): A ConstraintLayout containing:

EditText (For your command).

TextView (Preview of grabbed context).

Spinner (Model Selector).

ToggleButton (Reasoning Brain).

Button (Go/Magic Wand).

4. Technical Reference: The Context Grabber
This is the code that makes the "magic" happen. Put this inside your Service.

Kotlin
fun captureContextAndShowDialog() {
    val rootNode = rootInActiveWindow ?: return
    
    // 1. Find the field the user was typing in
    val focusedNode = rootNode.findFocus(AccessibilityNodeInfo.FOCUS_INPUT)
    
    // 2. Extract text (limit to last 500 chars for cost/speed)
    val currentText = focusedNode?.text?.toString() ?: ""
    val contextSnippet = currentText.takeLast(500)
    
    // 3. Store the node ID so we know where to paste later
    // (Note: Nodes can expire, so we might need to findFocus again before pasting)
    
    // 4. Show the Floating Dialog and pre-fill the context preview
    showPromptDialog(contextSnippet)
}

fun pasteResult(resultText: String) {
    // 1. Copy to Clipboard
    val clipboard = getSystemService(Context.CLIPBOARD_SERVICE) as ClipboardManager
    val clip = ClipData.newPlainText("ABoard AI", resultText)
    clipboard.setPrimaryClip(clip)
    
    // 2. Paste
    val rootNode = rootInActiveWindow ?: return
    val focusedNode = rootNode.findFocus(AccessibilityNodeInfo.FOCUS_INPUT)
    
    if (focusedNode != null) {
        focusedNode.performAction(AccessibilityNodeInfo.ACTION_PASTE)
    } else {
        Toast.makeText(this, "Copied to clipboard!", Toast.LENGTH_SHORT).show()
    }
}
